2. script元素

  • async ：立即下载脚本
  • charset：通过src属性指定代码的字符集
  • defer：脚本可以延迟到文档完全被解析显示后执行
  • language：语言（已废弃）
  • src：包含要执行代码的外部文件
  • type：代替language的

3. JavaScript嵌入方式

  1. 直接嵌入方式,在使用script元素嵌入JavaScript代码时，只需为script指定type属性，然后放入即可：
  2. 外部嵌入方式,如果要通过script元素来包含外部JavaScript文件，那么src属性就是必须的。这个属性的值是一个指向外部JavaScript文件的链接,注意如果包含了外部嵌入的JavaScript代码，则浏览器会优先下载外部的，而内部如果还包含JavaScript会被屏蔽掉。

4. 标签的位置

按照惯例，所有script元素都应该放在页面的元素中。这样是为了在head中把外部的文件都下载并执行完毕，但是这有时会造成延迟，所以现在的WEB应用程序会把JavaScript文件放到body中的内容之后，这样解析js之前，页面的内容将完全呈现在浏览器上。
也可以使用defer延迟脚本来避免这个问题，这样js外部代码会在遇到/html之后执行，但是defer只限于外部文件。

<script type="text/javascript" defer="defer" src="example.js">
</script>

5. 异步脚本

只适用于外部文件，并告诉浏览器立即下载文件。

<script type="text/javascript" async="async" src="example.js">
</script>

6. noscript

有些浏览器不支持js或者没启用js，所以noscript会在为启用js的时候显示，但是启用了js就不显示了。

7. 语法问题

  1. 一切变量区分大小写
  2. 标识符:第一个字符必须是字母、或者$，其他字符可是字母、、$或者数字，不能把关键字用作标识符。

8. 变量

1.js的变量是松散类型的，所谓松散类型就是可以用来保存任何类型的数据。例如：var message；其中message可以是任何类型的。像这样未进行初始化的变量，会保存一个特殊的值–undefined。var message=“hi”，这是一个字符型的初始值未hi的字符型变量。我们可以在修改变量值的同时修改变量的类型，但是一般不推荐这样做。

2.局部变量，定义在函数体内部的变量，变量在函数退出后会销毁变量。

function test（）{
var message = "hi!";
}
test();
alert(message);//错误

这段代码是错误的，因为message这个变量在函数借宿后被销毁了。

3.全局变量，定义为全局的变量，无伦在什么地方，都可以被使用。

function test(){
message = "hi!";
}
test();
alert(message);//hi!

    15. 变量复制

从一个变量向另一个变量复制基本类型值时，会在变量对象上创建一个新值，然后把该值复制到为新变量分配的位置上，意思就是复制后的新变量拥有者新的空间，和之前的互不影响。
从一个变量向另一个变量复制引用类型的值时，同样也会将存储在变量中的值复制一份放到为新变量分配的空间中。不同的是这个值的副本实际上是一个指针，而这个指针指向存储在堆中的一个对象。复制操作结束后，两个变量实际上将引用同一个对象，因此改变其中一个变量就会影响另一个变量。

var obj1 = new obbject()'
var obj2 = obj1;
obj1.name = "job";
alert(obj2.name);//值为job

18. 垃圾收集

标记清楚（最常用），当变量进入环境时，将这个变量标记为“进入环境”。而当变量离开时，将变量标记为“离开环境”。最后垃圾收集器完成内存清楚工作，销毁那些带标记的值并回收他们所占用的内存空间。
引用计数，跟踪每个值被引用的次数，当引用次数为0时，将其占用的内存空间收回，但是存在bug，循环引用时，两个值得计数永运不会为0.

19. 性能问题和内存管理

垃圾收集器什么时候运行决定着性能，ie浏览器之前的垃圾收集器是根据内存分配量来运行的，如果一个脚本包含很多变量，那么该脚本可能在生命周期内一直保存那么多变量，这样垃圾收集器就会频繁运行，导致性能问题严重，后来ie7改写了垃圾收集的例程：触发垃圾收集的变量分配、字面量和数组元素的临界值被调整为动态修正，如果低于15%就会加倍，如果回收了85%就会重置，这样极大地提升了性能。
优化内存的最好方式，就是为执行中的代码只保存必要的数据，一旦数据不再有用，最好通过将其值设置为null来释放其引用–这个方法叫解除引用。这一方法适用于大多数全局变量和全局对象的属性。局部变量会在他们离开执行环境时自动被解除引用。
